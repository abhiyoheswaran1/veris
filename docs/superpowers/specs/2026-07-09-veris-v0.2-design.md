# Veris v0.2 — Design Spec ("Developer Loop")

**Status:** Approved (design)
**Date:** 2026-07-09
**Branch:** `feat/veris-v0.2`
**Builds on:** v0.1 (shipped, tag `v0.1.0`). Reuses the `Runner`/orchestrator/verdict/reporter/evidence seams unchanged.

---

## 1. Thesis

Make Veris useful *during* active development: fast, change-aware feedback. v0.2 adds a
developer loop on top of v0.1's one-shot verification — without weakening the honesty
guarantee or the dependency discipline (runtime deps stay `cac` + `picocolors` only).

Two new commands:
- `veris affected` — run only the checks relevant to what changed.
- `veris watch` — re-run affected checks continuously as files change.

## 2. Locked decisions

| Decision | Choice | Rationale |
|---|---|---|
| `affected` depth | **Coarse: changed-files → gate the check set** (no import graph) | Useful immediately; keeps the real project/import graph as the v0.3 boundary |
| Changed-file baseline | **Working-tree vs `HEAD`** by default; `--base <ref>` override | Local loop wants uncommitted changes; CI/PR wants branch diff |
| Watch mechanism | **Native recursive `fs.watch`** + ~150ms debounce + git-diff reconcile; **`--poll`** mtime fallback | Zero new deps preserves dependency discipline; poll covers flaky-watch environments |
| Result cache | **In-memory, per watch session** | Cross-tick reuse without a persistent-cache design (deferred) |

## 3. Scope

### In
- `veris affected` and `veris watch` commands.
- Coarse change→check gating (`src/affected/gate.ts`).
- Git changed-file detection (`src/git/changes.ts`), working-tree-vs-`HEAD` default + `--base`.
- Native recursive `fs.watch` watcher with debounce + `--poll` fallback (`src/watch/watcher.ts`).
- Honest **scoped** verdicts with a distinct "not affected by changes" skip reason.
- Cross-tick in-memory result reuse in `watch` (unaffected checks show a "cached" marker).
- README/CHANGELOG updates; dogfood.

### Out (deferred — do NOT build in v0.2)
- Import/project graph and graph-based affected-test selection (**v0.3**).
- Persistent on-disk result cache.
- Slow-test surfacing, flaky-test detection, test-suite analytics (**v0.7**).
- HTML/JSON reporters (**v0.4**).
- `scan`, `plan`, `explain`, `review`, `generate`, risk scoring.

## 4. Architecture

New modules only; existing v0.1 modules are consumed unchanged.

```
src/
  git/
    changes.ts        # changedFiles(root, {base?}) -> string[] (via `git diff --name-only`)
  affected/
    gate.ts           # affectedChecks(files, project) -> CapabilityId[]
  watch/
    watcher.ts        # watch(root, {poll?, debounceMs?}, onBatch) -> stop()
  cli/commands/
    affected.ts       # runAffected(root, opts) -> exit code
    watch.ts          # runWatch(root, opts) -> never-returns (until SIGINT)
```

Data flow (`affected`): `changedFiles` → `affectedChecks` → `runChecks(project, ids, root)`
(v0.1 orchestrator, scoped to the affected ids) → `computeVerdict` (v0.1, already scopes
to the requested capabilities) → terminal + markdown reporters (v0.1).

`watch` wraps `affected`'s run logic: an initial run, then a debounced re-run per change
batch, retaining the last `CheckResult` per capability so unaffected capabilities render
from cache.

## 5. Interfaces

```ts
// src/git/changes.ts
export interface ChangeSet { files: string[]; base: string | null; } // base=null => working tree vs HEAD
export function changedFiles(root: string, opts?: { base?: string }): Promise<ChangeSet>;

// src/affected/gate.ts
export interface AffectedPlan {
  checks: CapabilityId[];        // capabilities to run (∩ available)
  reasonByCheck: Partial<Record<CapabilityId, string>>; // why each is affected
  changedCount: number;
}
export function affectedChecks(files: string[], project: Project): AffectedPlan;

// src/watch/watcher.ts
export interface WatchOptions { poll?: boolean; debounceMs?: number; }
export function watch(
  root: string,
  opts: WatchOptions,
  onBatch: (changedPaths: string[]) => void,
): () => void; // returns a stop() function
```

### Gating rules (`affectedChecks`)

Applied over the changed-file set; the union of matched capabilities is intersected with
`project` availability. **Conservative by default — when in doubt, run everything.**

| Changed file matches | Affects |
|---|---|
| `*.ts *.tsx *.mts *.cts` | types, lint, unit |
| `*.js *.jsx *.mjs *.cjs` | lint, unit |
| test files (`*.test.*`, `*.spec.*`, `test/**`, `__tests__/**`) | unit (+ lint) |
| config (`tsconfig*.json`, `biome.json*`, `.eslintrc*`, `eslint.config.*`, `package.json`, `veris.config.*`) | **all available** (config affects everything) |
| docs/assets only (`*.md`, images, `LICENSE`, etc.) | none |
| any other/unknown code file | **all available** (conservative fallback) |

`checks` is the union, intersected with `project.capabilities` that are `available`.

## 6. Honesty (carried from v0.1)

The core guarantee holds: a check that did not run is never reported as verified.

- `veris affected` prints a scoped header, e.g. `Affected  3 changed files → types, unit`,
  and computes the verdict over **only** the affected checks. v0.1's `computeVerdict`
  already scopes to the capabilities passed in — so an affected run of `[types, unit]`
  yields a verdict about `types`+`unit` only. Capabilities that exist but were **not
  affected** are surfaced with a distinct reason: **"not affected by changes"** (separate
  from "unavailable"/"not configured").
- **No relevant changes** → prints `Nothing affected — no checks run` and exits `0`. This
  is explicitly *not* a "Verified" (nothing was verified); the output must say so.
- Exit codes unchanged: `0` all affected checks passed, `1` a failure, `2` an affected
  capability was skipped/unknown (`--partial-ok` overrides), computed over the affected set.
- `watch` is a live view; its per-tick verdict is the affected-scoped verdict. Cached
  (unaffected) capabilities are marked `cached` and retain their last **real** status
  from when they actually ran; a `cached` pass is always shown as `cached`, never
  relabeled as a fresh pass.

## 7. Watch UX

- On start: one full affected run against the current working tree.
- On each debounced change batch (~150ms): recompute changed files, run affected checks,
  re-render the calm summary. Unaffected capabilities render with a `⟳ cached` marker and
  their last status.
- Ignores `.veris/`, `.git/`, `node_modules/`, `dist/`.
- Scrollback-friendly (no screen clear by default); `--clear` opts into clear-on-tick.
- `Ctrl-C` stops cleanly (calls the watcher's `stop()`).
- `--poll` switches to mtime polling for environments where recursive `fs.watch` is
  unreliable (e.g. some Linux/container setups).

## 8. Milestones (subagent-driven, TDD)

1. **`git/changes.ts`** — `changedFiles` (working-tree vs `HEAD`, `--base`, untracked), tests on a temp git repo.
2. **`affected/gate.ts`** — `affectedChecks` mapping incl. conservative fallbacks, table-driven tests.
3. **`veris affected` command** — scoped run + honest scoped output + "nothing affected" path + exit codes, tests.
4. **`watch/watcher.ts`** — native recursive `fs.watch` + debounce + `--poll` fallback + ignore rules, tests (real temp dir file events).
5. **`veris watch` command** — initial run + per-batch re-run + in-memory cross-tick reuse + `cached` rendering + clean SIGINT, tests.
6. **Docs + release** — README (watch/affected, the dev loop), CHANGELOG `0.2.0`, dogfood; bump `package.json` to `0.2.0`.

## 9. Risks & mitigations

| Risk | Mitigation |
|---|---|
| `fs.watch` recursive unreliable on some Linux/containers | `--poll` fallback; document it; debounce + git-diff reconcile so missed events self-heal on the next change |
| Editor atomic-save (rename) storms cause duplicate runs | Debounce batches events; reconcile against `git diff` (source of truth) rather than raw event paths |
| Coarse gating runs too much or too little | Conservative default (unknown/config → all); the verdict is honestly scoped either way, so "too much" is safe and "too little" is bounded by the explicit mapping |
| Affected "verified" misread as full verify | Scoped header + "not affected" reason + distinct wording; never a bare "Verified" |
| Watcher leaks/handles on exit | `watch()` returns `stop()`; command wires SIGINT to it |

## 10. Explicit non-goals for v0.2

Import/project graph and graph-based test selection (v0.3) · persistent disk cache ·
slow-test/flaky analytics (v0.7) · HTML/JSON reporters (v0.4) · `scan`/`plan`/`explain`/
`review`/`generate` · risk scoring. Anything here is refused in v0.2 with a clear message.
